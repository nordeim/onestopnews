Commitments

Built different.
By design, not by accident.

Every architectural choice exists for a documented, tested reason. These are not aspirational values — they are enforced by code, verified by audit, and visible to you.

01

Full AI Transparency

Every AI summary includes a Nutrition Label showing exactly what the model did, which sources it used, and what percentage of the original content it analyzed. Three machine-readable layers (JSON-LD, HTTP header, HTML meta) ensure automated compliance verification. This is not a feature — it is an invariant.

  • EU AI Act Article 50 compliant
  • 3-layer machine-readable provenance
  • Original source always one click away
02

Content Availability Guard

AI summarisation is only triggered when sufficient content exists. Articles with only a title or short excerpt are never summarised — doing so would require the AI to fabricate content. The content_availability enum enforces this at the database level.

  • 4-tier content availability classification
  • Database-enforced summarisation guard
  • Zero fabrication risk by architectural design
03

Sub-Second Feed Delivery

Cache Components with named profiles deliver a pre-rendered shell in under 800ms FCP. Cursor-based pagination eliminates full-table scans. The feed shell is static; only the article list streams in via Suspense boundaries. This is not caching luck — it is architectural intention.

  • FCP ≤800ms via PPR shell
  • Cursor pagination (no offset drift)
  • Streaming SSR with Suspense boundaries
04

Enterprise-Grade Reliability

BullMQ v5 job queue with priority lanes, dead-letter handling, and real-time monitoring. Source health tracked with failure counts and exponential backoff. Ingestion failures alert within 5 minutes of the third consecutive failure. Every pipeline is observable end-to-end.

  • ≤5min failure alerting
  • Exponential backoff for failing sources
  • End-to-end observability (Axiom / Sentry)
